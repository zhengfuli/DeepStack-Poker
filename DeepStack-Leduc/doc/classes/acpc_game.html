Class acpc_game

Handles communication to and from DeepStack using the ACPC protocol.

For details on the ACPC protocol, see http://www.computerpokercompetition.org/downloads/documents/protocols/protocol.pdf.

ACPCGame:connect (server, port)
Connects to a specified ACPC server which acts as the dealer.

Parameters:

  • server the server that sends states to DeepStack, which responds with actions
  • port the port to connect on

See also:

    network_communication.connect
ACPCGame:get_next_situation ()
Receives and parses the next poker situation where DeepStack must act.

Blocks until the server sends a situation where DeepStack acts.

Returns:

  1. the parsed state representation of the poker situation (see protocol_to_node.parse_state)
  2. a public tree node for the state (see protocol_to_node.parsed_state_to_node)
ACPCGame:play_action (adviced_action)
Informs the server that DeepStack is playing a specified action.

Parameters:

  • adviced_action

    a table specifying the action chosen by Deepstack, with the fields:

    • action: an element of constants.acpc_actions

    • raise_amount: the number of chips raised (if action is raise)
