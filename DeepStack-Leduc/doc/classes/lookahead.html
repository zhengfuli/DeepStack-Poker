Lookahead:resolve_first_node (player_range, opponent_range)
Re-solves the lookahead using input ranges.

Uses the input range for the opponent instead of a gadget range, so only appropriate for re-solving the root node of the game tree (where ranges are fixed).

build_lookahead must be called first.

Parameters:

  • player_range a range vector for the re-solving player
  • opponent_range a range vector for the opponent
Lookahead:resolve (player_range, opponent_cfvs)
Re-solves the lookahead using an input range for the player and the CFRDGadget to generate ranges for the opponent.

build_lookahead must be called first.

Parameters:

  • player_range a range vector for the re-solving player
  • opponent_cfvs a vector of cfvs achieved by the opponent before re-solving
Lookahead:get_chance_action_cfv (action_index, board)
Gives the average counterfactual values for the opponent during re-solving after a chance event (the betting round changes and more cards are dealt).

Used during continual re-solving to track opponent cfvs. The lookahead must first be re-solved with resolve or resolve_first_node.

Parameters:

  • action_index the action taken by the re-solving player at the start of the lookahead
  • board a tensor of board cards, updated by the chance event

Returns:

    a vector of cfvs
Lookahead:get_results ()
Gets the results of re-solving the lookahead.

The lookahead must first be re-solved with resolve or resolve_first_node.

Returns:

    a table containing the fields:

    • strategy: an AxK tensor containing the re-solve player's strategy at the root of the lookahead, where A is the number of actions and K is the range size

    • achieved_cfvs: a vector of the opponent's average counterfactual values at the root of the lookahead

    • children_cfvs: an AxK tensor of opponent average counterfactual values after each action that the re-solve player can take at the root of the lookahead
