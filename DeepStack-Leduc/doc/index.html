DeepStack for Leduc Hold'em and variants

Modules

data_generation Generates neural net training data by solving random poker situations.
data_generation_call Generates neural net training data by evaluating terminal equity for poker situations.
random_card_generator Samples random card combinations.
evaluator Evaluates hand strength in Leduc Hold'em and variants.
card_to_string_conversion Converts between string and numeric representations of cards.
card_tools A set of tools for basic operations on cards and sets of cards.
net_builder Builds the neural net architecture.
arguments Parameters for DeepStack.
constants Various constants used in DeepStack.
game_settings Game constants which define the game played by DeepStack.
train Trains the neural network.
tools Assorted tools.

Scripts

main_data_generation Script that generates training and validation files.
cpu_gpu_model_converter Generates a neural net model in CPU format from a neural net model saved in GPU format.
deepstack Performs the main loop for DeepStack.
main_train Script that trains the neural network.

Classes

acpc_game Handles communication to and from DeepStack using the ACPC protocol.
network_communication Handles network communication for DeepStack.
protocol_to_node Converts between DeepStack's internal representation and the ACPC protocol used to communicate with the dealer.
range_generator Samples random probability vectors for use as player ranges.
bet_sizing Gives allowed bets during a game.
cfrd_gadget Uses the the CFR-D gadget to generate opponent ranges for re-solving.
lookahead A depth-limited lookahead of the game tree used for re-solving.
lookahead_builder Builds the internal data structures of a Lookahead object.
mock_resolving Implements the re-solving interface used by resolving with functions that do nothing.
resolving Implements depth-limited re-solving at a node of the game tree.
bucket_conversion Converts between vectors over private hands and vectors over buckets.
bucketer Assigns hands to buckets on the given board.
masked_huber_loss Computes a Huber loss for neural net training and evaluation.
mock_nn_terminal Implements the same interface as value_nn, but without uses terminal equity evaluation instead of a neural net.
next_round_value Uses the neural net to estimate value at the end of the first betting round.
value_nn Wraps the calls to the final neural net.
continual_resolving Performs the main steps of continual re-solving, tracking player range and opponent counterfactual values so that re-solving can be done at each new game state.
terminal_equity Evaluates player equities at terminal nodes of the game's public tree.
data_stream Handles the data used for neural net training and validation.
strategy_filling Fills a game's public tree with a uniform strategy.
tree_builder Builds a public tree for Leduc Hold'em or variants.
tree_cfr Runs Counterfactual Regret Minimization (CFR) to approximately solve a game represented by a complete game tree.
tree_strategy_filling Recursively performs continual re-solving at every node of a public tree to generate the DeepStack strategy for the entire game.
tree_values Computes the expected value of a strategy profile on a game's public tree, as well as the value of a best response against the profile.
tree_visualiser Generates visual representations of game trees.
